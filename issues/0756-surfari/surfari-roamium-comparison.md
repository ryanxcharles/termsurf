# Surfari / Roamium Matrix Comparison

This artifact accounts for every scenario accepted by
`scripts/ghostboard-geometry-matrix.sh` and maps it to Surfari evidence.

Classifications:

- `Equivalent` — Surfari has direct evidence for the same user-visible behavior.
- `Covered by focused Surfari harness` — the exact Roamium scenario name is not
  reused, but the behavior is directly proven by Surfari-specific harnesses.
- `Engine-specific difference` — the behavior is intentionally different because
  Surfari uses WebKit instead of Chromium/Roamium.
- `Not applicable` — the scenario tests Roamium launch/resolver behavior,
  Ghostboard/WebTUI-only behavior, performance budget, or another path outside
  Surfari engine parity.
- `Gap` — Surfari lacks direct evidence or fails the comparable behavior.

## Summary

No `Gap` rows remain for Surfari engine parity. The scenarios classified as
`Not applicable` are Roamium launch/resolver paths, Ghostboard/WebTUI-only UI
behavior, AppKit/window behavior outside the browser engine, or performance
thresholds outside Issue 756's correctness matrix.
