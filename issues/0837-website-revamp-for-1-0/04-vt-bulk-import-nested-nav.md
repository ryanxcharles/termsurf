# Experiment 4: VT bulk import + nested navigation (Phase 1)

## Description

Experiment 3 stood up the VT hosting infrastructure (the static `VTSequence`
component, MDX wiring, attribution, footnote styling) and proved it with a
4-page slice. This experiment **bulk-imports the remaining ~60 Ghostty VT MDX
files** (mechanically) and builds the **nested "Terminal API" navigation** the
full set needs (which also resolves the single-level section-ordering limitation
deferred from Experiment 1).

**Scope (decomposed after the second design review):** this experiment is the
**mechanical foundation** — import all 64 pages, fix links/anchors, inject
frontmatter, build nested nav — keeping product claims **upstream-attributed**
so no unverified TermSurf assertion ships. The user's chosen TermSurf rebrand
(with per-claim fork verification and a per-file adversarial accuracy pass) is
delivered by **Experiment 5+**, one per VT subsection (decision 7). This keeps
each experiment's Pass genuinely true instead of partially-true.

### The corpus (surveyed)

64 files total under Ghostty's `docs/vt` (MIT): `index.mdx`, `reference.mdx`,
`external.mdx`, and `concepts/` (4), `control/` (5), `csi/` (29), `esc/` (8),
`osc/` (15). Four are already imported (Exp 3 slice: `index`, `csi/cup`,
`control/bel`, `osc/7`). Of the 64, **52 contain internal links / `#TODO` /
reference-style link defs** and **16 mention "Ghostty"** in prose.

### Key decisions

1. **Mechanize the import with a committed-output script
   (`scripts/import-vt.ts`).** Like `gen-references.ts`, it reads from a Ghostty
   website checkout (path via `--in`/env), copies all 64 files into
   `src/content/docs/vt/**` preserving the directory structure, and applies the
   mechanical adaptations below. Output is committed; a `--check` mode flags
   drift. Re-running re-imports cleanly (idempotent). The 4 slice files are
   regenerated by the script too, so they stop being hand-maintained — the
   script becomes the single source of truth for all VT pages.
2. **Frontmatter injection.** Each file gets `section: "Terminal API"`,
   `subsection` (Concepts / Control / CSI / ESC / OSC; the three top-level pages
   `index`/`reference`/`external` get no subsection), an `order` (alphabetical
   within subsection, with `index` first), and `navLabel` (the short sequence
   name, e.g. "CUP", derived from the title's parenthetical or the filename).
   `index` keeps `navLabel: Overview`.
3. **Link adaptation (mechanical, with fragment validation).** All VT pages are
   imported, so VT-internal links resolve 1:1 when paths are preserved. Rules:
   `[text](#TODO)` **and bare `[text](#)`** placeholders → plain `text` (the
   review found `csi/ech.mdx:14` uses a bare `#`, which the `#TODO`-only rule
   missed); Ghostty config-reference links (`/docs/config/reference...`) →
   `/docs/reference/config` (drop anchor); reference-style defs
   (`[label]: /docs/...`) → same remap, else inline; `/docs/vt/...` links kept
   (they build). **Fragment links must resolve to real heading slugs** — the
   review found `reference.mdx`'s `osc/conemu#change-progress-state-(osc-94)`
   does not match the `github-slugger` output `change-progress-state-osc-94`, so
   the importer normalizes/fixes such anchors. The build-time crawl validates
   internal links **including `#fragment` targets against each page's emitted
   heading ids**, not just page existence.
4. **Voice (interim): upstream-attributed, no unverified TermSurf claims.** The
   user chose "rebrand to TermSurf **and** verify every product claim against
   the fork", gated by a per-file adversarial accuracy pass. The second review
   showed that doing the rebrand and the verification in one 64-file experiment
   creates a contradiction — mechanically rebranding all claims while verifying
   only some would ship unverified TermSurf assertions as fact (e.g.
   `concepts/colors.mdx` "Ghostty currently supports three…" → "TermSurf
   currently supports three"). So this experiment is scoped to the **mechanical
   foundation** and keeps product/behavior **claims upstream-attributed** (they
   are true about Ghostty, and TermSurf's terminal is a Ghostty fork) behind a
   clear section-level framing note; the **full TermSurf rebrand + fork
   verification is deferred to Experiment 5+** (decision 7), which carries the
   user's decisions. Concretely the importer applies only **safe** transforms:
   rename `## Ghostty Status` → `## Implementation Status`; leave external
   upstream URLs untouched (`concepts/cursor.mdx:54`); and **does not** assert
   any TermSurf-specific product fact. No claim is rewritten from "Ghostty" to
   "TermSurf" in this experiment — that only happens after fork verification.
   This makes verification §3 ("no unverified TermSurf claim ships") trivially
   true here and defers the hard accuracy work to gated follow-ups where each
   page's Pass is genuinely earned.
5. **Nested navigation (the real change).** Extend the content schema with an
   optional `subsection: string`, and extend `src/lib/docs-nav.ts` + the sidebar
   to render **two levels**: a section heading, then per-subsection groups. To
   keep a ~60-item section usable with **zero JS**, render each VT subsection as
   a native `<details>`/`<summary>` disclosure (collapsible without JavaScript).
   **The `<details>` containing the current page renders `open`** so the active
   item is visible (review finding). Under "Terminal API", the subsection-less
   pages (`index`/`reference`/`external`) render as **flat items first**, then
   the subsection disclosures — this mixed layout is specified, not incidental.
   Sections without subsections (Components, Protocol, Reference) render flat,
   exactly as today (backward compatible). Subsection order is explicit
   (Concepts, Control, CSI, ESC, OSC) via a small ordering map, which also
   **resolves the deferred section/subsection-ordering limitation** (no more
   reliance on alphabetical `localeCompare`).
6. **Scope guard.** Importer + schema + nav + styles only. No Ghostboard fork
   _source_ changes (the fork is **read** to verify claims, per decision 4). The
   `reference.mdx` page (Ghostty's hand-curated, admittedly partial sequence
   index) is imported with links/anchors adapted; extending its coverage is out
   of scope (scope decision 1 is "reuse as base, then extend" — extension is
   later/ongoing).
7. **TermSurf rebrand + fork-verified accuracy — deferred to Experiment 5+
   (carries the user decisions).** The user's "rebrand + verify every claim" +
   "per-file adversarial accuracy pass" is delivered as a series of follow-up
   experiments, **one per VT subsection** (Concepts, Control, CSI, ESC, OSC,
   then the top-level pages). Each: rebrands that subsection's claims to
   TermSurf, verifies every product/behavior claim against the **whole fork**
   (`ghostboard/src/**` — not just `terminal/**`; the flagship "~15s timeout" is
   in `apprt/gtk/class/surface.zig`, and platform-specific behaviors must be
   verified **for the platform the claim implies**, with TermSurf being
   macOS-primary), sets compatibility-table TermSurf cells from the fork,
   softens or removes unverifiable specifics, and passes a per-file adversarial
   accuracy review before its result commit. Splitting this out (per the second
   design review) keeps each experiment's Pass genuinely true rather than
   partially-true.

## Changes

Files in `website/` unless noted:

1. **`scripts/import-vt.ts`** (new) — the importer: copy + frontmatter inject +
   **mechanical** link/anchor adapt + safe voice rebrand + claim-flagging +
   `--check`, reading the Ghostty website checkout (`--in <dir>` / env; the user
   passes the checkout — the website repo isn't vendored). Writes all 64 `vt/**`
   pages and emits a list of flagged product/behavior claims for the accuracy
   gate.
2. **`src/content/docs/vt/`** (generated, committed) — all 64 VT pages
   (replacing the 4 hand-imported slice files). The committed files are the
   importer's **pure mechanical output**, so `--check` is **byte-exact** (the
   `gen-references.ts` precedent). Accuracy edits do not happen here — they are
   Experiment 5+ content (decision 7), at which point those pages are no longer
   regenerated by this importer (declared explicitly when that work starts).
3. **`src/content.config.ts`** — add `subsection: z.string().optional()`.
4. **`src/lib/docs-nav.ts`** — nested grouping (section → subsection), explicit
   section + subsection ordering maps, active-subsection detection.
5. **`src/components/DocPage.astro`** — render nested nav; VT subsections as
   `<details>`/`<summary>` disclosures, `open` on the active subsection;
   subsection-less pages as flat items first.
6. **`src/styles/style.css`** — styles for the nested/disclosure nav.
7. **`package.json`** — `"import:vt": "bun run scripts/import-vt.ts"`.
8. **`website/CLAUDE.md`** — document the importer and that VT pages are
   mechanically generated/committed with claims upstream-attributed pending the
   Experiment 5+ rebrand/verification (supersedes the Exp 3 "slice only" note).

   `#TODO`/bare-`(#)` placeholder.

     or dangling fragment.
3. **No unverified TermSurf claims (interim invariant).** Because this

   TermSurf-specific product fact: the only safe transform applied is the

   upstream-attributed under the section framing note. (The TermSurf rebrand +
   fork verification is Experiment 5+.)

     TermSurf-specific claim not yet fork-verified.

## Design Review

**Pass 1 — REJECT.** An independent `adversarial-reviewer`, checking the real
corpus, rejected the original design: a blanket mechanical "Ghostty" →
"TermSurf" prose swap asserts **unverified TermSurf-specific facts** (e.g.
`osc/conemu.mdx`'s "~15s timeout", `osc/22.mdx`'s CSS cursor names,
`reference.mdx`'s "supports many more sequences", `concepts/colors.mdx` channel
limits), and verifying only the _absence of the word_ "Ghostty" checks the wrong
property — a swap to a false claim would pass. It also found the link rules miss
bare `(#)` placeholders (`csi/ech.mdx:14`) and don't validate `#fragment`
targets (`reference.mdx`'s `conemu#change-progress-state-(osc-94)` ≠ the
`github-slugger` slug), the compat-table handling was under-specified
(multi-terminal tables, Yes/No matrices, not just version columns), and the
nested-nav design omitted the active-page `open` state and the mixed
flat+disclosure layout.

**User decisions** (the rejection surfaced a genuine content-voice fork):

- **Voice:** rebrand to TermSurf **and verify every product claim against the
  Ghostboard fork** — not the lighter "keep Ghostty + framing note" option.
- **Accuracy gate:** a **per-file adversarial accuracy pass** (not build-checks
  only).

**Pass 2 — APPROVE WITH CHANGES.** The reviewer confirmed the four pass-1
blockers resolved (verified against the corpus: the `csi/ech.mdx:14` bare `(#)`,
the `reference.mdx` conemu fragment mismatch, the zero-JS nested `<details>`),
but found three more issues, all resolved by **decomposing the work** (its
recommendation):

1. **Contradiction** — rebranding all claims then verifying only some would ship
   unverified TermSurf assertions while recording "Pass". Resolved: this
   experiment no longer rebrands claims at all; it keeps them
   upstream-attributed (decision 4), and the rebrand+verification moves to
   Experiment 5+ (decision 7) so each Pass is genuinely earned.
2. **Fork-source scope** — claims span beyond `terminal/**` (the "~15s timeout"
   is in `apprt/gtk/class/surface.zig:1106`, and is GTK-only while TermSurf is
   macOS-primary). Resolved: decision 7 widens verification to
   `ghostboard/src/**` with platform-appropriate checking.
3. **`--check` incoherence** — resolved by committing the importer's **pure
   mechanical output** (byte-exact `--check`, the `gen-references` precedent);
   accuracy edits are Experiment 5+ content, not post-edits here.

Also folded in: verification §1 asserts the 64-file count (60 new + 4
regenerated) so a partial import can't pass on a stale subset.

This experiment is now the mechanical foundation; the user's rebrand +
fork-verified + per-file-accuracy decisions are carried by the Experiment 5+
subsection passes. The decomposition was the reviewer's own recommendation, so
applying it satisfies the APPROVE-WITH-CHANGES condition; implemented on that
basis.

## Result

**Result:** Pass

The full Ghostty-parity VT corpus is imported with clean links and the nested
Terminal API navigation; all eight verification criteria pass. The site builds
**76 pages** (12 prior + 64 VT). Product claims remain upstream-attributed (Exp
5+ rebrands).

### One bug found and fixed during implementation

The first `Ghostty Status` rename regex used `\s*$`; `\s` matched the newline
and collapsed the blank line after the heading. Fixed to `[ \t]*$` (verified the
blank line is preserved). Caught before commit.

## Completion Review

Independent `adversarial-reviewer` at the result gate. **Verdict: APPROVE.** It
reproduced every claim against the repo, a fresh build, and the upstream source:
`--check` exits 0; 76 pages build; `astro check` 0 errors; 64/64 files imported
filename-for-filename with only the intended adaptations; an independent
head+body crawl found 0 dead links / dangling fragments / placeholders (incl.
the re-slugged conemu anchor); the interim-voice invariant holds (the only
"TermSurf" in VT source is the `index` framing note — no product claim was
rebranded, e.g. `osc/22` CSS cursor names, `concepts/colors` channel limits,
`conemu` "~15s timeout" remain byte-identical upstream); nested nav renders
correctly with active-open on deep pages and zero JS; commit separation is
correct.

Two **optional** findings; both addressed after review:

- **`concepts/screen.mdx` shipped `<meta description="TODO">`** (upstream is a
  stub). Fixed: the importer now drops a `TODO` description.
- **VT pages had no visible `<h1>`** (not in the reviewer's criteria, found in a
  post-review self-check): Ghostty's layout renders the frontmatter title, but
  TermSurf's `DocPage` does not, so all 64 VT pages were headless while every
  other doc page has a body `# Title`. Fixed: the importer now prepends
  `# {title}` to each page (also giving each a heading anchor id). Verified
  (`<h1 id="cursor-position-cup">Cursor Position (CUP)</h1>`), `--check` still
  byte-exact, build still 76 pages / 0 errors / 0 dead links.

Deferred (cosmetic): OSC alphabetical-by-filename ordering → numeric-aware in
the Exp 5+ OSC pass.
