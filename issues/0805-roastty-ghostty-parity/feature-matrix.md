# Feature Matrix

This matrix tracks user-visible and integration-visible feature/workflow parity
between pinned Ghostty and Roastty.

## Row Schema

| Column                     | Meaning                                                         |
| -------------------------- | --------------------------------------------------------------- |
| ID                         | Stable row ID, prefixed `FEAT-`.                                |
| Domain / Feature           | User-visible feature, workflow, or integration behavior.        |
| Upstream behavior / source | Ghostty behavior, source path, or workflow reference.           |
| Roastty behavior / path    | Roastty behavior, source path, or workflow reference.           |
| Status                     | `Pass`, `Gap`, `Intentional divergence`, or `Not applicable`.   |
| Verification method        | Concrete test, command, walkthrough, or oracle.                 |
| Evidence artifact          | Log, screenshot, test output, or matrix row proving the status. |
| Guard tier                 | Tier 0-4 from the Issue 805 regression guard policy.            |
| Guard command / checklist  | Exact command or manual checklist that catches regressions.     |
| Run cadence                | When the guard should run.                                      |
| Guard sufficiency          | Why this guard is strong enough for the row.                    |
| Owner experiment           | Experiment that created or last updated the row.                |
| Notes                      | Short context, if needed.                                       |
