# Issue 617: Alpha

## Goal

Remove all legacy version tags and version display from the website. TermSurf
has been rebuilt from scratch multiple times (ts1 through gui) and the old 1.0
version tags are misleading. The project is in alpha — not 1.0.

## Background

The repo carries 344 git tags. Most are inherited from upstream dependencies
(WezTerm, cef-rs) that were imported via subtree or vendored directly. The
TermSurf-specific tags (`v0.1.0` through `v1.2.3`) refer to ts1-era releases
that no longer represent the current application. No one ever used those
releases externally.

The website at termsurf.com displays a version badge (`v1.0.0`) in the header,
sourced from `website/data/version.json` which is generated from the latest git
tag by `website/scripts/build-version.ts`. Once all tags are deleted, this
pipeline will break — and we want it to. The version display should be removed
entirely.

| Misc              | `nightly*`, `tip`, `pre_blink_merge`, etc.      | ~5    | WezTerm                        |

All 344 tags should be deleted — locally and on the GitHub remote.

### Website version display

The version badge is rendered in `website/src/components/Header.tsx` next to the
TermSurf title. It reads from `website/data/version.json`, which is generated by
`website/scripts/build-version.ts` (runs `git describe --tags --abbrev=0`).

## Proposed changes

1. **Delete all local tags** — `git tag -l | xargs git tag -d`
2. **Delete all remote tags** — `git push origin --delete $(git tag -l)` (or
   equivalent batch operation)
3. **Remove version display from Header.tsx** — Delete the version badge span
4. **Remove version build script** — Delete `website/scripts/build-version.ts`
   and `website/data/version.json`
5. **Remove `build:version` from package.json** — Clean up the build pipeline

## Experiments

### Experiment 1: Delete all tags and remove version display

Delete all 344 git tags (locally and on the `origin` remote), then remove the
version display pipeline from the website.

#### Changes

**Git operations (no file changes):**

1. Delete all local tags: `git tag -l | xargs git tag -d`
2. Delete all remote tags on `origin`:
   `git ls-remote --tags origin | awk '{print $2}' | xargs -n1 git push origin --delete`

Only `origin` (github.com/termsurf/termsurf) is affected. The `upstream`
(ghostty), `wezterm`, and `cef-rs` remotes are untouched.

**`website/src/components/Header.tsx`:**

- Remove the `import versionData` line
- Remove the version badge `<span>` from the header
- Remove `relative inline-block` from the `<h1>` (no longer needed for absolute
  positioning)

**`website/package.json`:**

- Remove the `build:version` script
- Change `build:data` from `bun run build:commits && bun run build:version` to
  `bun run build:commits`

**Delete files:**

- `website/scripts/build-version.ts`
- `website/data/version.json`

#### Verification

1. `git tag -l` returns empty
2. `git ls-remote --tags origin` returns empty
3. `website/src/components/Header.tsx` has no version badge
4. `bun run build:data` in `website/` succeeds (only builds commits)
5. Tags on `upstream`, `wezterm`, and `cef-rs` remotes are unaffected

**Result:** Pass

All 344 local tags deleted. All 26 remote tags on `origin` deleted. Version
badge, build script, and data file removed from the website. The `build:data`
pipeline now only builds commits.

## Conclusion

TermSurf is alpha software. The legacy version tags (`v0.1.0` through `v1.2.3`)
and inherited upstream tags (WezTerm, cef-rs) have been purged — 344 local and
26 remote. The website no longer displays a version number. When TermSurf is
ready for a real release, new tags will be created from scratch.
