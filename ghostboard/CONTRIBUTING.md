# Contributing to Ghostboard

Ghostboard development happens inside the TermSurf repository.

For project workflow, use the root `../AGENTS.md` instructions and the issue
documents in `../issues/`. Significant work should be described by an issue and
implemented through one experiment at a time.

This fork does not use Ghostty's contributor-gate process. Do not follow or
recreate upstream Ghostty contribution-policy automation for local TermSurf
work.

For local build notes, see `HACKING.md`.
