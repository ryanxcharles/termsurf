<h1>Configuration</h1>

<p>
  TermSurf uses Ghostboard's Ghostty-based configuration system. This page
  covers the current config file location and settings relevant to TermSurf.
</p>

<h2>Config File</h2>

<p>TermSurf looks for config at:</p>

<ol>
  <li><code>~/.config/termsurf/config</code></li>
</ol>

<p>Minimal config example:</p>

<pre class="bg-background-dark p-3 border-l-2 border-border text-foreground-dark text-xs mb-6 overflow-x-auto"><code><span class="text-muted"># ~/.config/termsurf/config</span>
font-size = <span class="text-success">14</span>
theme = <span class="text-success">TokyoNight</span>
window-decoration = <span class="text-success">false</span></code></pre>

<h2>TermSurf-Relevant Settings</h2>

<h3>window-decoration</h3>

<p>
  Controls the window title bar and frame. Default is <code>true</code>
  (standard macOS title bar).
</p>

<pre class="bg-background-dark p-3 border-l-2 border-border text-foreground-dark text-xs mb-6 overflow-x-auto"><code><span class="text-muted"># No title bar</span>
window-decoration = <span class="text-success">false</span></code></pre>

<p>
  With <code>false</code>, the title bar is hidden and the content view fills the
  entire window. Browser overlays align correctly in both modes.
</p>

<h3>font_size</h3>

<p>
  Font size affects cell dimensions, which determine the pixel size and position
  of browser overlays. Changing the font size resizes all overlays.
</p>

<h3>color_scheme</h3>

<p>
  Ghostty-compatible color themes apply to terminal panes. Browser content uses
  its own color scheme, controlled via the <code>:dark</code> command in the Web
  TUI.
</p>

<h2>Browser Profiles</h2>

<p>Browser profiles are stored at:</p>

<pre class="bg-background-dark p-3 border-l-2 border-border text-foreground-dark text-xs mb-6 overflow-x-auto"><code>~/.local/share/termsurf/chromium-profiles/<span class="text-accent">&lt;profile-name&gt;</span>/</code></pre>

<p>
  Each profile gets its own Chromium user data directory with separate cookies,
  storage, and cache. Profiles are specified per-pane when launching the
  <code>web</code> TUI.
</p>

<h2>Data Directories</h2>

<table>
  <thead>
    <tr><th>Path</th><th>Purpose</th></tr>
  </thead>
  <tbody>
    <tr>
      <td><code>~/.config/termsurf/</code></td>
      <td>Config files</td>
    </tr>
    <tr>
      <td><code>~/.local/share/termsurf/</code></td>
      <td>Application data</td>
    </tr>
    <tr>
      <td><code>~/.local/share/termsurf/chromium-profiles/</code></td>
      <td>Browser profile data</td>
    </tr>
    <tr>
      <td><code>~/.local/state/termsurf/</code></td>
      <td>Log files</td>
    </tr>
    <tr>
      <td><code>$TMPDIR/termsurf/</code></td>
      <td>Unix sockets, runtime files</td>
    </tr>
  </tbody>
</table>

<h2>Socket Paths</h2>

<p>Ghostboard listens on a PID-scoped socket:</p>

<pre class="bg-background-dark p-3 border-l-2 border-border text-foreground-dark text-xs mb-6 overflow-x-auto"><code>$TMPDIR/termsurf/termsurf-ghostboard-<span class="text-accent">&lt;pid&gt;</span>.sock</code></pre>

<p>
  This path is exposed to child processes via the <code>TERMSURF_SOCKET</code>
  environment variable. Each browser engine process also listens on its own
  socket:
</p>

<pre class="bg-background-dark p-3 border-l-2 border-border text-foreground-dark text-xs mb-6 overflow-x-auto"><code>$TMPDIR/termsurf/roamium-<span class="text-accent">&lt;hash&gt;</span>-<span class="text-accent">&lt;gui-pid&gt;</span>-<span class="text-accent">&lt;profile&gt;</span>.sock</code></pre>
