<h1>Keybindings</h1>

<p>
  This page covers the <strong>terminal</strong> key bindings configured in your
  TermSurf config file with the <code>keybind</code> option. Ghostboard inherits
  Ghostty's keybinding system, so the syntax below matches Ghostty. For the full
  list of actions you can bind, see the{" "}
  <a href="/docs/reference/keybind-actions">Keybind Actions</a> reference.
</p>

<blockquote>
  <p>
    These are the terminal's bindings, not the <code>web</code> browser TUI's.
    The <a href="/docs/components/webtui">Web TUI</a> has its own modal,
    vim-inspired keys for driving the browser; this page does not cover those.
  </p>
</blockquote>

<h2>Format</h2>

<p>
  A binding has the form <code>keybind = trigger=action</code>. Set it in your
  config (see the <a href="/docs/reference/configuration">configuration
  guide</a> for the file location):
</p>

<pre class="bg-background-dark p-3 border-l-2 border-border text-foreground-dark text-xs mb-6 overflow-x-auto"><code><span class="text-muted"># ~/.config/termsurf/config</span>
keybind = <span class="text-success">ctrl+z=close_surface</span>
keybind = <span class="text-success">ctrl+d=new_split:right</span></code></pre>

<p>
  Duplicate triggers overwrite earlier ones. Triggers are not unique per prefix
  — <code>ctrl+a</code> and <code>global:ctrl+a</code> (see{" "}
  <a href="#prefixes">Prefixes</a>) overwrite each other.
</p>

<h2>Triggers</h2>

<p>
  A trigger is a <code>+</code>-separated list of keys and modifiers, for
  example <code>ctrl+a</code>, <code>ctrl+shift+b</code>, or <code>up</code>.
</p>

<h3>Modifiers</h3>

<p>The valid modifiers (with their aliases) are:</p>

<ul>
  <li>
    <code>shift</code>
  </li>
  <li>
    <code>ctrl</code> (alias <code>control</code>)
  </li>
  <li>
    <code>alt</code> (aliases <code>opt</code>, <code>option</code>)
  </li>
  <li>
    <code>super</code> (aliases <code>cmd</code>, <code>command</code>)
  </li>
</ul>

<p>
  A modifier cannot repeat (<code>ctrl+ctrl+a</code> is invalid), and a trigger
  may contain only one key (<code>ctrl+a+b</code> is invalid). Modifiers and the
  key may appear in any order.
</p>

<h3>Unicode keys vs. physical keys</h3>

<p>
  A single character such as <code>a</code> matches by its unmodified codepoint,
  which depends on the keyboard layout, and matching is case-insensitive
  (<code>ctrl+A</code> matches <code>ctrl+a</code>). To match a physical key
  position regardless of layout, use a{" "}
  <a href="https://www.w3.org/TR/uievents-code/">W3C key code</a> such as{" "}
  <code>KeyA</code> (a lowercase snake-case form like <code>key_a</code> also
  works), or the <code>physical:</code> trigger prefix to force physical-key
  matching. Physical keys always match at higher priority than Unicode
  codepoints. The special key <code>catch_all</code> matches any key that is not
  otherwise bound.
</p>

<h2>Sequences (leader keys)</h2>

<p>
  Join multiple triggers with <code>&gt;</code> to require a sequence — what
  other software calls a leader key or key chord. There is no limit on the
  number of steps.
</p>

<pre class="bg-background-dark p-3 border-l-2 border-border text-foreground-dark text-xs mb-6 overflow-x-auto"><code><span class="text-muted"># Press ctrl+a, then n</span>
keybind = <span class="text-success">ctrl+a&gt;n=new_split:right</span></code></pre>

<p>
  TermSurf waits indefinitely for the next key in a sequence (there is no
  timeout). If you set a sequence as a CLI argument, quote it, because{" "}
  <code>&gt;</code> is special in most shells. Binding a prefix directly (e.g.{" "}
  <code>ctrl+a</code>) unbinds any sequences that started with it.
</p>

<h2>Actions</h2>

<p>
  An action is either <code>action</code> or <code>action:param</code> (the
  parameterized form is used when the action takes a value). The full list is in
  the <a href="/docs/reference/keybind-actions">Keybind Actions</a> reference. A
  few special actions are worth calling out:
</p>

<ul>
  <li>
    <code>ignore</code> — do nothing (black-hole the input).
  </li>
  <li>
    <code>unbind</code> — remove the binding so the key passes through to the
    program.
  </li>
  <li>
    <code>csi:…</code>, <code>esc:…</code>, <code>text:…</code> — send a CSI
    sequence, an escape sequence, or a literal string.
  </li>
</ul>

<p>
  The special value <code>keybind = clear</code> removes <em>all</em> bindings
  set up to that point, including the defaults.
</p>

<h2 id="prefixes">Prefixes</h2>

<p>A trigger can be prefixed to change how the binding behaves:</p>

<ul>
  <li>
    <code>all:</code> — apply the binding to every terminal surface, not just
    the focused one (only affects surface-specific actions).
  </li>
  <li>
    <code>global:</code> — make the binding work system-wide, even when TermSurf
    is not focused (supported on macOS). It implies <code>all:</code>.
  </li>
  <li>
    <code>unconsumed:</code> — do not consume the input; the encoded key is also
    sent to the running program. (<code>global:</code> and <code>all:</code>
    bindings always consume.)
  </li>
  <li>
    <code>performable:</code> — only consume the input if the action actually
    runs (e.g. <code>copy_to_clipboard</code> with no selection does nothing and
    passes the key through).
  </li>
</ul>

<p>
  Sequences are not allowed with <code>global:</code> or <code>all:</code>{" "}
  triggers.
</p>
