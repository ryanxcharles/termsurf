<h1>Roamium</h1>

<p>
  Roamium is the Chromium browser engine binary for TermSurf. It wraps
  <code>libtermsurf_chromium</code> (a C shared library built from the Chromium
  fork) in a small Rust binary (~400 lines) that handles Unix socket IPC,
  protobuf message dispatch, and process lifecycle.
</p>

<h2>How it works</h2>

<p>
  The GUI (<a href="/docs/architecture">Ghostboard</a>) spawns one Roamium
  process per browser profile. Each process serves exactly one profile — one set
  of cookies, storage, and cache — a hard constraint of Chromium's
  <code>BrowserContext</code>. The GUI passes the IPC socket, the profile's
  user-data directory, and the engine's own listen socket on launch; Roamium
  enters Chromium's message loop, creates the profile's context, registers with
  the GUI, and then creates tabs and streams GPU layers on demand. For the wire
  format and topology, see <a href="/docs/architecture">Architecture</a>; for
  the message types, see the{" "}
  <a href="/docs/protocol/messages">protocol message reference</a>.
</p>

<h2>Architecture</h2>

<p>
  Roamium follows a two-layer pattern designed to be reusable across browser
  engines:
</p>

<ul>
  <li>
    <strong>
      <code>libtermsurf_chromium</code>
    </strong>{" "}
    — a C shared library that wraps Chromium's embedding API. Exports the{" "}
    <code>ts_*</code> functions below for lifecycle, profiles, tabs, navigation,
    input, and state. Built from the Chromium fork.
  </li>
  <li>
    <strong>Roamium</strong> — a Rust binary that links the C library, connects
    to the GUI socket, dispatches incoming protobuf messages to
    <code>ts_*</code> calls, and relays Chromium callbacks back out as protobuf
    messages.
  </li>
</ul>

  The auth and dialog callbacks are request/reply round trips: the

  , which collects the answer and replies via <code>ts_reply_http_auth</code> /
  <code>ts_reply_javascript_dialog</code>.
</p>

<h2>Multi-engine pattern</h2>

<p>
  Roamium is the Chromium engine binary, and Surfari is the WebKit engine
  binary. Each engine pairs a C shared library wrapping the engine's embedding
  API (the <code>ts_*</code> functions) with the reusable ~400-line Rust binary
  — only the C library changes per engine. Future researched engines include
  Waterwolf (Gecko) and Girlbat (Ladybird) — see the
  <a href="/docs/roadmap">roadmap</a>.
</p>

<h2>Installation</h2>

<p>
  Homebrew installs Roamium to
  <code>/opt/homebrew/opt/termsurf-roamium/</code>. That root holds the
  <code>roamium</code> binary, the Chromium dylibs, <code>.pak</code> resources,
  ICU data, V8 snapshots, and generated resources under <code>gen/</code>. The
  GUI resolves the installed engine at
  <code>/opt/homebrew/opt/termsurf-roamium/roamium</code>. See{" "}
  <a href="/docs/getting-started">Getting Started</a> to install.
</p>
