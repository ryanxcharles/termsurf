<h1>Roadmap</h1>

<p>
  These are <strong>planned</strong> beyond the current macOS release. TermSurf
  1.4 ships with Roamium (Chromium) and Surfari (WebKit); see
  <a href="/docs/about">About</a> for what's available today. The items below
  describe the project's direction, with no committed dates.
</p>

<h2>Browser features</h2>

<ul>
  <li>Bookmarks</li>
  <li>Multi-tab browsing (a user-facing tabbed UI)</li>
  <li>History</li>
  <li>Downloads</li>
  <li>PDF viewing</li>
</ul>

<h2>Browser engines</h2>

<p>
  The protocol is engine-agnostic, so more engines are an extension rather than
  a rewrite. Roamium (Chromium) and Surfari (WebKit) ship today; Gecko
  (Waterwolf) and Ladybird (Girlbat) are <strong>researched</strong> — under
  consideration, not yet committed. See the engine table in{" "}
  <a href="/docs/architecture">Architecture</a> for current status.
</p>

<h2>Terminal front-ends</h2>

<p>
  TermSurf's protocol lets any terminal emulator host browser overlays. Beyond
  Ghostboard (the current GUI), planned front-ends include Kitty, Alacritty, and
  iTerm2.
</p>

<h2>Platforms</h2>

<p>
  TermSurf currently ships on macOS. Linux and Windows are planned; iOS and
  Android may follow.
</p>
