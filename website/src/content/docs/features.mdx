<h1>Features</h1>

<p>
  Because Ghostboard is a Ghostty fork, TermSurf inherits Ghostty's terminal
  features. This page is an overview of the ones available on macOS today; for
  the exact configuration options, see the{" "}
  <a href="/docs/reference/config">config reference</a>.
</p>

<h2>Color themes</h2>

<p>
  The <code>theme</code> option selects a color theme — a built-in theme, a
  custom theme name, or a path to a theme file. You can also set different
  themes for light and dark mode. TermSurf ships with a large collection of
  built-in themes; run <code>termsurf +list-themes</code> to browse them, and
  drop your own theme files in <code>~/.config/termsurf/themes</code>.
</p>

<pre class="bg-background-dark p-3 border-l-2 border-border text-foreground-dark text-xs mb-6 overflow-x-auto"><code><span class="text-muted"># ~/.config/termsurf/config</span>
theme = <span class="text-success">TokyoNight</span></code></pre>

<p>
  A theme file is just another config file, so it can set any color option. See{" "}
  <a href="/docs/reference/config">the <code>theme</code> option</a> for the
  full details and the light/dark syntax.
</p>

<h2>Shell integration</h2>

<p>
  Shell integration auto-injects support for <code>bash</code>,{" "}
  <code>fish</code>, <code>elvish</code>, <code>nushell</code>, and{" "}
  <code>zsh</code>. It enables a number of conveniences:
</p>

<ul>
  <li>New tabs and splits inherit the previous pane's working directory.</li>
  <li>
    Prompt marking, which powers the <code>jump_to_prompt</code> keybinding.
  </li>
  <li>Closing a pane at the prompt doesn't ask for confirmation.</li>
  <li>Cleaner redraws when resizing with a complex prompt.</li>
</ul>

<p>
  It's controlled by <code>shell-integration</code> (default{" "}
  <code>detect</code>) and <code>shell-integration-features</code>, which toggles
  individual features such as <code>cursor</code>, <code>title</code>,{" "}
  <code>sudo</code>, and <code>path</code>. See{" "}
  <a href="/docs/reference/config">the shell-integration options</a> for the
  full list and syntax. Prompt-related actions can also be bound — see{" "}
  <a href="/docs/reference/keybindings">Keybindings</a>.
</p>

<h2>SSH integration</h2>

<p>
  Two shell-integration features smooth out SSH sessions:{" "}
  <code>ssh-env</code> keeps environment variables compatible with remote hosts
  (adjusting <code>TERM</code> so remote programs behave even when they don't
  know TermSurf's terminfo), and <code>ssh-terminfo</code> installs the
  terminal's terminfo entry on remote hosts automatically. Once installed on a
  host, it's cached; the <code>termsurf +ssh-cache</code> CLI action manages that
  cache. Enable them via <code>shell-integration-features</code> — see{" "}
  <a href="/docs/reference/config">the config reference</a>.
</p>

<h2>AppleScript automation (macOS)</h2>

<p>
  TermSurf ships an AppleScript scripting dictionary, so you can automate it from
  AppleScript or <code>osascript</code>. The dictionary exposes{" "}
  <code>application</code>, <code>window</code>, <code>tab</code>, and{" "}
  <code>terminal</code> objects, plus commands to open and arrange the
  terminal — including <code>new window</code>, <code>new tab</code>,{" "}
  <code>split</code>, <code>focus</code>, and <code>close</code> — and to drive
  it: <code>input text</code>, <code>send key</code>, the mouse commands, and{" "}
  <code>perform action</code> (which runs any keybind action by name).
</p>

<pre class="bg-background-dark p-3 border-l-2 border-border text-foreground-dark text-xs mb-6 overflow-x-auto"><code><span class="text-muted"># Open a new TermSurf window from the shell</span>
<span class="text-primary">osascript</span> -e <span class="text-success">'tell application "TermSurf" to new window'</span></code></pre>

<p>
  Because <code>perform action</code> accepts any keybind action, scripts can
  trigger the same operations you bind to keys — see the{" "}
  <a href="/docs/reference/keybind-actions">Keybind Actions</a> reference.
</p>
